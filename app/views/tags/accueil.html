Bienvenue sur le StarterKi Ausy pour Play!Framework

Si vous lisez cette page c'est que vous avez correctement installé Play!Framework et réussi à lancer le StarterKit Ausy
Ce StarterKit a bien sur pour objectif packager quelques classes et modules intéressants afin de vous aidez à commencer une application Play! mais il contient également un support de formation se basant sur le mini-projet AusyPlay accessible sur github git://github.com/kebuu/AusyPlay.git (mais ça vous devriez déjà le savoir).

Si vous connaissez déjà bien Play!Framework et que le support de formation ne vous intéresse pas, vous pouvez supprimer tous ce qui concerne cette documentation et ne garder qu'un StarterKit vide prêt à l'emploi. Dans le cas contraire il est conseillé de récupérer le projet support pour pouvoir démarrer l'application et voir un peu comment tout cela fonctionne.

Note Afin que vous puissiez démarrer en même temps le projet AusyPlay et le StarterKit, le port du StarterKit a été positionné sur 9001 (et le port JPDA sur 8001), ce qui implique que pour accéder à l'application StarterKit vous dévrez utiliser l'adresse http://localhost:9001

Contenu

Ce StarterKit est basé sur la version 1.2.4 de Play!Framework (en RC3 au moment de rédaction de ces lignes) et il contient les éléments suivants :

Le module CRUD
Ce module permet de générer simplement des pages de visualisation, création, modification et suppression de vos objets model
Le module Secure
Ce module permet créer un page d'identification et de gérer des profiles utilisateurs pour sécuriser l'accès à votre application
Le module Guice
Ce module permet d'utiliser Guice pour gérer l'injection de dépendance dans votre projet
La version 1.5.2 de JQuery
JQuery est une librairie javascript assez géniale, si vous ne savez pas de quoi il s'agit, rendez-vous sur JQuery
La feuille de style Bootstrap Twitter
Cette feuille de style vous permet de donner simplement et rapidement un peu d'allure à votre application. Pour apprendre à l'utiliser, rendez-vous sur Bootstrap Twitter. Cette page documentation utilise quelques éléments de cette feuille de style.
La classe JPQLHelper
Cette classe, développée en interne vous aidera à réaliser vos requêtes JPQL dynamiques en gérant notamment le cas des paramètres nuls ou vides.
Le support de formation basé sur le projet AusyPlay
Vous trouverez en plus des explications globales, des exemples de classes de test unitaires et fonctionnels, des Jobs, des classes de base pour vos models ou vos controlleurs, un exemple d'envoi de mail ou d'utilisation de WebSocket ...

Objectifs

L'objectif de ce StaterKit est donc double :

  1. Proposer quelques outils supplémentaires pour pouvoir démarrer votre projet encore plus rapidement.
  2. Proposer une présentation de Play!Framework ainsi qu'un support de formation basé sur le projet AusyPlay
