Le module CRUD

Le module CRUD est un module packagé avec la distribution de Play! et permettant de générer automatiquement des pages CRUD pour vos objets model. Pour l'activer il suffit, comme le module Secure de le déclarer dans le fichier de dépendance dependencies.yml : 

require:
    - play 1.2.3
    - play -> guice 1.2
    - play -> crud
    - play -> secure

Ensuite, pour chaque objet model pour lesquels vous souhaitez avoir la génération de page CRUD vous devez creer un controlleur ayant les caractéristiques suivantes :

  1. Votre controlleur doit étendre la classe CRUD
  2. Votre controlleur doit avoir pour nom le même nom que la classe model que vous voulez exposer suivi d'un "s"

Voici un exemple de controlleur CRUD pour la classe Serie : 

@With(Secure.class)
public class Series extends CRUD {

}

Note : L'annotation @With(Secure.class) indique simplement que l'utilisateur doit être connecté pour accéder à ce controlleur, sinon il sera redirigé vers la page de login. Il s'agit en faite de l'utilisation du module Secure.

Avant de pouvoir tester le module CRUD il ne vous reste plus qu'à importer dans les routes du modules (comme on l'avait fait pour le module Secure), pour cela ajouter la ligne suivante dans votre fichier routes : 

*      /crud                                        module:crud

Il ne vous reste plus qu'à pointer sur l'url /crud.

Personnalisation : il est possible de personnaliser les pages du module CRUD. Une première solution est de penser à redéfinir la méthode toString() de vos objets model. Vous pouvez également modifier les templates CRUD. Pour savoir comment faire rendez-vous sur la documentation de Play! et pour avoir un exemple de personnalisation regarder les pages app > views > Users > list.html et app > views > Series > show.html
